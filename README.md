## 📂 데이터

### 사용 데이터셋

| 파일 | 설명 | 행 수 |
| `bus_data.csv` | 서울 25개 자치구별 버스 정류소 개수 | 25행 |
| `cctv_data.csv` | 서울 25개 자치구별 CCTV 설치 개수 | 25행 |
| `crime_data.csv` | 서울 25개 자치구별 범죄 유형 및 건수 | 950행 |

### 데이터 클린징 과정

1. 컬럼명 표준화— CSV 원본의 공백이 포함된 컬럼명을 SQLite 테이블에 저장 시 `버스정류소개수`, `CCTV개수` 등으로 일관되게 정리했습니다.
2. JOIN 기준 통일 — 3개 데이터셋 모두 `자치구` 컬럼을 기준으로 병합하며, 이름 불일치 없이 25개 구가 모두 매칭됩니다. 이를 위해 모든 데이터에 자치구 칼럼이 존재하도록 하였고, 서울 강남시, 강남, 강님시 등을 강남시로 통일했습니다. 
4. 집계 처리 — `crime_data.csv`는 범죄 대분류·중분류별로 세분화된 데이터로, 분석 시 `SUM(건수)`로 자치구별 총 범죄 건수를 집계했습니다.
5. 파생 변수 생성 — 정류소당 범죄율(`총범죄건수 / 버스정류소개수`), CCTV당 범죄율(`총범죄건수 / CCTV개수`) 등 비율 지표를 SQL 내에서 직접 산출했습니다.

## 📊 시각화 결과

### 1️⃣ 교통 편의(버스 정류소) vs 범죄율

- 이중축 막대그래프 — 버스 정류소 상위 10개 자치구의 정류소 수(좌축)와 총 범죄 건수(우축)를 나란히 비교합니다.
- 산점도 + 추세선 — 전체 25개 구의 정류소 수와 총 범죄 건수를 산점도로 표현하고, 선형 추세선과 상관계수(r)를 함께 제시합니다.
- 분류 표 — 중위값 기준으로 "교통 편리 & 범죄 적음(추천 지역)"과 "교통 불편 & 범죄 많음(주의 지역)"을 자동 분류합니다.

> 💡 인사이트: 버스 정류소가 많아 유동인구가 집중되는 자치구일수록 범죄 노출 빈도가 다소 높아지는 경향이 있습니다. 그러나 교통 편의와 범죄율이 반드시 비례하지는 않으며, 일부 구는 교통이 편리하면서도 낮은 범죄율을 유지하고 있습니다.

---

### 2️⃣ CCTV 개수 vs 범죄율

- 막대 + 꺾은선 이중축 그래프 — CCTV 상위 10구에서 설치 개수(막대)와 총 범죄 건수(꺾은선)의 관계를 직관적으로 비교합니다.
- 산점도 + 추세선 — 전체 25구의 CCTV 수와 총 범죄 건수 분포를 산점도로 확인합니다.
- 치안 효과 지역 표 — CCTV가 충분히 많으면서 동시에 범죄 건수가 낮은 자치구를 자동으로 추출하여 제시합니다.

> 💡 인사이트: CCTV가 많다고 범죄가 무조건 줄어들지는 않으며, 범죄 발생이 잦은 지역에 CCTV를 집중 배치하는 역방향 관계도 관찰됩니다. 그러나 CCTV를 선제적으로 확대한 자치구에서 상대적으로 낮은 범죄율이 유지되는 패턴도 확인됩니다.

---

### 3️⃣ 버스 정류소 · CCTV · 범죄 3자 상관관계

- 버블 차트 — 버스 정류소 수(x축)와 CCTV 수(y축)를 기준으로, 버블의 크기와 색상이 총 범죄 건수를 나타냅니다.
- 상관계수 히트맵 — 3개 변수 간의 상관계수를 Seaborn 히트맵으로 시각화합니다.
- 4분면 산점도 — 버스 정류소 수(x)와 총 범죄 건수(y)를 평균값 기준으로 4분면(교통↑범죄↓ / 교통↑범죄↑ / 교통↓범죄↑ / 교통↓범죄↓)으로 구분하며, CCTV 수를 색상으로 표현합니다.
- 지표 카드 — 버스↔범죄, CCTV↔범죄, 버스↔CCTV 상관계수 3개를 요약 카드로 제공합니다.

> 💡 **인사이트:** '교통 편의 → 유동인구 증가 → CCTV 확충 → 범죄 억제'라는 선순환 구조가 일부 자치구에서 작동하고 있음을 확인할 수 있습니다. 교통과 CCTV가 모두 잘 갖춰진 자치구에서 범죄율이 낮게 유지되는 경향이 가장 뚜렷하게 나타납니다.
